# REQ-04: 정적 사이트로 방향 전환

- 요청일: 2026-05-19
- 상태: ✅ 완료 (2026-05-19)
- 스프린트: 7

## 요청 원문

> "이거 계획을 좀 변경하고 싶어. 지금 로그인, 백엔드로 예약관리 이런거 다 지워줘. 무조건 번호로만 예약할거야"

## 핵심 목표

- 사이트는 정비소 소개·연락처 안내 역할만
- 손님은 사이트의 전화번호로 직접 전화해 예약
- 백엔드·DB·로그인·관리자 페이지 모두 폐기
- 운영 비용 = 도메인비만 (연 1.5만원)

## 사용자 시나리오

1. 손님이 사이트 진입 → 매장 소개·서비스·갤러리·영업시간·전화번호 확인
2. "예약하기" 또는 전화 아이콘 클릭 → 모바일에서 즉시 통화 화면(tel: link)
3. 점주는 손님 전화 받아 일정 확인 (사이트 시스템 없음)

## 기능 목록

| # | 기능 | 우선순위 | 설명 | 상태 |
|:--|:--|:--|:--|:--|
| F-1 | 홈 페이지 | 필수 | Hero + 서비스 + 갤러리 + 영업시간 + 지도 + 후기 + CTA. 모두 정적 데이터 | 미시작 |
| F-2 | 예약 안내 페이지 | 필수 | "전화로 예약하기" 큰 버튼 + 영업시간 + 위치 | 미시작 |
| F-3 | 정적 데이터 단일 출처 | 필수 | `frontend/src/data/shop.ts` 한 파일 | 미시작 |
| F-4 | 백엔드 폐기 | 필수 | `backend/` 디렉토리 + Postgres + Alembic 삭제 | 미시작 |
| F-5 | 프론트 정리 | 필수 | Login/Admin/Lookup/useAuth 등 삭제 | 미시작 |

## 기술 결정

- **프레임워크**: Vite + React 유지 (이미 구축됨, 정적 빌드 지원)
- **데이터 출처**: `frontend/src/data/shop.ts` 단일 파일
- **호스팅 권장**: Cloudflare Pages / Vercel / Netlify (정적 무료)
- **CMS**: 도입 안 함 (점주가 직접 코드 수정 또는 향후 재논의)
- **알림톡/OAuth**: 모두 폐기
