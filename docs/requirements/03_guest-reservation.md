# REQ-03: 비회원 예약 + 회원/비회원 통합 폼

- 요청일: 2026-05-12
- 상태: ❌ 폐기 (Sprint 7, 2026-05-19 — 시나리오 X 전환)
- 스프린트: 2

## 요청 원문

> 카톡 로그인이랑 구글 로그인 네이버 로그인 살리고 이걸 회원 예약, 비회원예약은 정보입력 후 예약으로 하자.
> q2(비회원 예약 폼): 너가 말한거 하고
> q3(비회원 조회/취소): B로 하자 (전화번호 + 예약번호로 조회/취소 가능)

## 핵심 목표

- 로그인하지 않아도 정보 입력 후 예약 가능 (비회원 예약)
- 회원 / 비회원 모두 **동일한 폼**으로 예약
- 비회원도 **전화번호 + 예약번호**로 본인 예약을 조회/취소 가능

## 사용자 시나리오

1. 로그인 안 한 방문자가 예약 페이지에서 이름/전화번호/차량/일정/메모 입력 후 예약 → 예약번호를 안내받음
2. 로그인한 회원이 동일 폼을 보지만 이름·전화번호가 본인 정보로 자동 채워짐 (수정 가능)
3. 비회원이 나중에 예약 조회 페이지에서 전화번호 + 예약번호로 본인 예약을 확인하거나 취소
4. 관리자 ERP 화면에는 회원/비회원 예약이 통합된 목록으로 보임 (비회원은 user_id 없이 customer_name/phone으로 표시)

## 기능 목록

| # | 기능 | 우선순위 | 설명 | 상태 |
|:--|:--|:--|:--|:--|
| F-1 | Reservation 모델 변경 | 필수 | `user_id` nullable, `customer_name`/`customer_phone` 컬럼 추가 | 미시작 |
| F-2 | 통합 예약 생성 | 필수 | `POST /api/reservations` (Optional Auth). 비로그인 시 customer_* 필수 | 미시작 |
| F-3 | 회원 자동 채움 | 필수 | 회원 로그인 시 폼 초기값에 본인 이름·전화 자동 입력 | 미시작 |
| F-4 | 비회원 예약 조회 | 필수 | `GET /api/reservations/lookup?phone=&id=` (간단 인증) | 미시작 |
| F-5 | 비회원 예약 취소 | 필수 | `DELETE /api/reservations/{id}?phone=` (간단 인증) | 미시작 |
| F-6 | 어드민 통합 목록 | 필수 | 회원/비회원 예약 동일 목록. 회원이면 user 이름, 비회원이면 customer_name 표시 | 미시작 |
| F-7 | 예약번호 안내 화면 | 필수 | 예약 직후 예약번호 노출 + "조회는 전화번호 + 예약번호로" 안내 | 미시작 |
| F-8 | 비회원 조회 페이지 | 필수 | `/reservation/lookup` 라우트, 전화번호 + 예약번호 입력 폼 | 미시작 |

## 기술 결정 (PM-08 확정)

- **예약번호**: `lookup_code String(6) UNIQUE NOT NULL` — 영숫자 6자리 (헷갈리는 0/O/1/I/L 제외). `secrets.choice` 사용
- **비회원 조회 인증**: `phone + lookup_code` 매칭 + IP 기준 rate limit (5분 윈도우 10회 실패 시 차단). in-memory 카운터
- **폼 데이터 정합성**: 회원/비회원 모두 폼 입력값(`customer_name`/`customer_phone`)을 Reservation에 그대로 저장. User 정보는 건드리지 않음. 대리 예약 케이스 허용
- **비회원 취소**: hard delete 아닌 `status=rejected`로 변경 (이력 보존)
- **rate limit**: IP 기준, 프록시 뒤에서는 `X-Forwarded-For` 첫 번째 값 사용
